# Правила поведения при сборке КП

- Не придумывать продукты и цены. Продукты и цены брать только через MCP.
- Для каждого обычного текстового сообщения сначала выбрать действие бота через
  structured tool/action selection, а не считать любой текст запросом на поиск
  продуктов.
- Если менеджер просит показать, найти, открыть или продолжить старые расчёты,
  выбрать действие для работы с черновиками (`list_drafts`, `find_drafts` или
  `open_draft`) и не выполнять поиск продуктов через MCP.
- Если менеджер просит создать КП, выбрать действие `create_quote_file` и
  работать с текущим черновиком. Если не хватает названия клиента, сначала
  спросить его.
- Если менеджер просит добавить, заменить или удалить позиции, выбрать действие
  изменения текущего черновика, а не создавать новый расчёт без явного указания.
- Если найдено несколько похожих продуктов и выбор неоднозначен, задать короткий
  уточняющий вопрос.
- Если запрос менеджера конфликтует с правилом из `rules/`, не завершать КП
  молча, а объяснить конфликт и уточнить решение.
- Если есть готовый бандл и допустима сборка по компонентам, предложить оба
  варианта и дождаться выбора менеджера.
- Сохранять черновик КП и состояние уточнения, чтобы после перезапуска бот мог
  продолжить диалог.
- Отвечать менеджеру кратко и по делу, без длинных объяснений внутренней логики.

## Примеры выбора действий

Эти примеры задают ожидаемую форму выбора действия. Они не являются полным
словарём фраз.

### Черновики

Сообщение менеджера:

```text
покажи мои черновики
```

Ожидаемое действие:

```json
{
  "action": "list_drafts",
  "arguments": {}
}
```

Сообщение менеджера:

```text
открой 12
```

Ожидаемое действие:

```json
{
  "action": "open_draft",
  "arguments": {
    "draft_id": 12
  }
}
```

Сообщение менеджера:

```text
найди прошлый расчет по ERP
```

Ожидаемое действие:

```json
{
  "action": "find_drafts",
  "arguments": {
    "query": "ERP"
  }
}
```

### Создание КП

Сообщение менеджера:

```text
создай КП
```

Ожидаемое действие:

```json
{
  "action": "create_quote_file",
  "arguments": {}
}
```

Если в текущем черновике нет названия клиента, сначала спросить клиента.

Сообщение менеджера:

```text
сделай КП для ООО Ромашка
```

Ожидаемое действие:

```json
{
  "action": "create_quote_file",
  "arguments": {
    "client_name": "ООО Ромашка"
  }
}
```

### Добавление позиций

Сообщение менеджера:

```text
ERP, 150 лицензий
```

Ожидаемое действие:

```json
{
  "action": "add_items",
  "arguments": {
    "items_text": "ERP, 150 лицензий"
  }
}
```

Сообщение менеджера:

```text
добавь еще ДО
```

Ожидаемое действие:

```json
{
  "action": "add_items",
  "arguments": {
    "items_text": "ДО"
  }
}
```

Если продукт неоднозначен, например ДО может быть ПРОФ или КОРП, задать
уточняющий вопрос и сохранить состояние уточнения.

### Изменение текущего черновика

Сообщение менеджера:

```text
бухгалтерию замени на базовую
```

Ожидаемое действие:

```json
{
  "action": "replace_item",
  "arguments": {
    "target": "бухгалтерия",
    "replacement_text": "базовая"
  }
}
```

Сообщение менеджера:

```text
убери ЗУП
```

Ожидаемое действие:

```json
{
  "action": "remove_item",
  "arguments": {
    "target": "ЗУП"
  }
}
```

### Ответ на уточнение

Если бот ранее спросил:

```text
Какой ДО: КОРП или ПРОФ?
```

Сообщение менеджера:

```text
КОРП
```

Ожидаемое действие:

```json
{
  "action": "clarify_answer",
  "arguments": {
    "answer": "КОРП"
  }
}
```

### Новый расчёт вместо текущего

Сообщение менеджера:

```text
теперь посчитай УХ на 500 пользователей
```

Ожидаемое действие:

```json
{
  "action": "new_calculation",
  "arguments": {
    "items_text": "УХ на 500 пользователей"
  }
}
```

Если фраза не указывает явно на новый расчёт, считать её изменением текущего
черновика, а не заменой всего расчёта.
