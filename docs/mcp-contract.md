# MCP-контракт со стороны бота

## Назначение

Бот работает с внешним MCP-сервером `mcp-1c-price` через adapter layer. Этот
документ описывает не внутреннюю схему MCP-сервера, а минимальный контракт,
который нужен боту. MCP client может мапить фактический ответ сервера в эти
структуры.

## Lifecycle

- Бот запускает MCP-сервер как subprocess при старте приложения.
- Путь к серверу берётся из `MCP_SERVER_PATH`.
- Транспорт v1: `stdio`.
- Если MCP недоступен при старте или во время вызова, бот сообщает менеджеру
  понятную ошибку.
- При ошибке MCP Quote service не сохраняет частично изменённый черновик.
- Следующий запрос к ценам должен пересоздать MCP client/subprocess.

## Инструменты

### `search_products`

Вход:

```json
{
  "query": "ERP сервер x64",
  "limit": 10
}
```

Выход adapter:

```json
{
  "items": [
    {
      "code": "2900000000000",
      "name": "Название продукта",
      "price_retail": 100000,
      "vat": "НДС 20%"
    }
  ]
}
```

### `get_product`

Вход:

```json
{
  "code": "2900000000000"
}
```

Выход adapter:

```json
{
  "code": "2900000000000",
  "name": "Название продукта",
  "price_retail": 100000,
  "vat": "НДС 20%"
}
```

### `build_quote`

Вход:

```json
{
  "items": [
    {
      "code": "2900000000000",
      "qty": 2
    }
  ]
}
```

Выход adapter:

```json
{
  "items": [
    {
      "code": "2900000000000",
      "name": "Название продукта",
      "qty": 2,
      "price_retail": 100000,
      "vat": "НДС 20%",
      "line_sum": 200000
    }
  ],
  "total_sum": 200000
}
```

### `refresh_prices`

Вход:

```json
{}
```

Выход adapter:

```json
{
  "status": "updated",
  "message": "База цен обновлена"
}
```

`status` в v1 — display/status text для ответа менеджеру; бизнес-логика бота не
должна зависеть от конкретного набора значений, кроме успешного или ошибочного
выполнения MCP-вызова.
